Sjöstugan

Booking & Management System

A custom fullstack management system for Sjöstugan Bed & Breakfast. The system features a multistep booking form for guests handles guest bookings and... .

Role:

Full stack development

Scope:

Logic Mapping, Database schema, UI/UX

Platform:

Node.js, PostgreSQL, Docker

Final desktop homepage

Problems

Client side

Customers needed an intuitive, responsive booking experience that provided instant availability feedback.

The primary challenge was in designing a multi-step interface that simplifies complex information, maintains the brand identity, and has a secure connection with the database in order to avoid double bookings and errors.

Admin side

The owners needed a simple, centralized, and password protected platform to store and organize bookings.

The primary challenge was to create a secure management dashboard allowing real-time CRUD operations. Ensuring the owners can find, create, update, and delete bookings in a time-efficient and secure manner.

Client Booking Form

I implemented a multi-step state to ensure guest selections are preserved throughout the session. Each step has real-time front-end validation to ensure only valid information reaches the database.

The review step allows guests to double check their information before submission

The booking form maintaings the brand identity for visual continuity in the user experience for guests navigating from the main Sjöstugan website. A progress bar was implements allowing the guests to know the number of steps remaining at all times.

Admin dashboard

The admin dashboard lets the owners quickly scan "Todays status". Specific guests are easily found in the list of bookings with filters for booking timeframe and specific cabins. The search function allows for specific guest names to be searched for.

The admin pages feature a simplified version of the brand style as quick information scannability is top priority.

Features
  • Quick display of current occupancy
  • Search for guest to find booking
  • Filter bookings by
  • Filter bookings by cabin
  • Edit existing booking
  • Delete booking
  • Create new booking
  • Edit price of cabins

Database handling

Guest bookings and cabin information is stored in a PostgreSQL database that is containerized in a Docker container. All edit the admin make on the page are updated in real-time in the database.

Bcrypt password hashing ensures the admin credentials are never stored in plain text.

I implemented server side session management using Expess-sessions. Allowing the admin to stay logged in when making changes.

Project outcome

The management systems allows the owners to save hours on manual email correspondece. The ability to edit bookings eliminates the need to create entirely new bookings for minor guest input errors.

The Realtime connection with the database allows users to see which dates are available for booking and eliminates double bookings.
